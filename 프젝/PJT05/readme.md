## 이번 pjt를 통해 배운 내용

- N:M의 관계에 대해 생각해보고 관계를 설정한 후 역참조와 참조에 대해 생각하는게 어려웠다. 관계를 ForeignKey를 통해 설정하여 참조 관계를 만드는 것은 어렵지 않았으나, 역참조를 하거나 참조를 할 때 다소 헷갈리는 부분이 있었다. 이 부분은 N:M 모델을 계속 만들어보면서 직접 하다 보면 자연스럽게 배울 수 있다고 생각한다.

- 이번 프로젝트의 핵심 구현은 댓글 기능이었다. detail 페이지에 댓글을 구현하려면 user 와 댓글, 댓글과 게시글, 게시글과 유저가 서로 참조가 되어야 했다. 이러한 참조를 통해 자신이 작성한 댓글만 삭제 할 수 있거나 각 게시물에 id 를 참조하여 각 게시글마다 댓글을 달 수 있었다.

- 그 외에 수업에서 배웠던 팔로잉 팔로워 기능과 좋아요 기능도 구현하려 했지만 다음에 나만의 프로젝트를 진행할 때 해봐야겠다. 

- 이번 프로젝트에서도 장고 공식문서를 잘 활용하지 못했다. 장고 공식문서와 챗 gpt를 같이 활용하니 그나마 좀 활용할 만 했다. 

- Movie model 작성시 스크롤로 장르를 선택하는 부분을 구현하는데 지피티의 도움을 받았다.

```python
    GENRE_CHOICES = [
```

## 느낀점

- 장고 프로젝트를 이렇게 마무리 하게 되며, 확실히 프로젝트를 처음부터 내가 만들어서 진행하니 디버깅 할 때 더욱 편했다. CRUD를 어느정도 이해를 해서 그런지 전에 진행했던 프로젝트보다 좀 더 원할하게 프로젝트를 완성 시킬 수 있었다. 

- 하지만 장고 마지막에 배운 json 파일을 활용한 장고는 거의 이해를 하지 못했다. 크롤링이나 api를 통해 얻은 정보를 활용하기 위해서는 장고를 더 공부해야 한다고 느꼈다.

- 이렇게 어느정도 구성을 갖춘 페이지를 만들었지만, 장고의 유연하고 많은 기능을 제공해주는 '프레임워크' 로서의 기능 덕분에 만들었지 spring이나 다른 프레임워크로 만들기 위해서는 이 기능이 어떻게 동작하고 어떤 조건이 필요하는지 더 자세히 공부 할 필요성을 느꼈다. 그렇게 공부를 하게 된다면 어떤 프레임 워크가 나오고 어떤 프레임 워크를 사용하든 기본적인 구조가 머리에 있다면 더 쉽고 더 다양하게 활용할 수 있을 거라고 생각했다.

- 확실히 성장하고, 많이 능숙해졌지만 아직 더 많은 걸 배우고 공부해야 한다고 생각한다. 물론 현업에서 사용하는 개발 툴이나 언어 등 많은 것이 다르겠지만 기본적인 역량을 채워서 어떤 언어나 개발 기술이든 쉽게 활용할 수 있는 기반을 다져야겠다. 그렇게 기반을 쌓은 개발자가 되기 위해 오늘도 열심히 공부를 해야겠다. '모든 것은 기본에서 시작한다' 라는 손웅정 선생님의 말 처럼 항상 머리에 새기고 자만하지 않고 기본이 확실한 개발자로 성장하겠다.
