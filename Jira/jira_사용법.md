## 1. 에자일
디자인, 개발, 테스트 라는 스프린트를 여러번에 걸쳐서 진행

> 일정표 중심이 아닌 일을 중심으로 관리하는 것. 백로그와 일의 우선순위를 나눠서 관리

이를 우선 순위를 따지고 이를 백로그로 관리하고 어떤 일을 이번 차수(스프린트)에 반영 할 것인가

### 장점
- 기능을 **작게 구현**
- 바로 배포하는 **이터레이션을 반복**함으로써
- 고객에게 **지속적인 가치**를 제공하는 개발방식

## 3. 지라
- 에픽 : 큰 범위의 목표 -> 여러 스프린트에 걸쳐 완성됨

    - 스토리, 테스크 : 스프린트 내로 완성하기로 한 **약속**

        - 서브 테스크 : 스토리 내의 디테일한 과제


<br>

## 실전 지라 활용팁
- 스프린트 시작 후 백로그 아래 쪽에 이슈 추가 가능  

- 에픽을 추가하여 큰 범위의 카테고리를 넣으면 됌  

- 스토리 포인트를 0~4 포인트 넣을 수 있음  

- 스토리의 담당자를 지정 가능  

- 백로그에서 스프린트로 옮겨서 활성화 가능  

- 업무를 시작하기 전에 진행중으로 옮기기  


## 실습코치님의 피보나치 수열을 활용한 지라 포인트 관리

1) 피보나치수열을 응용한 0, 1, 2, 3, 5, 8, ... 과 같은 수열을 이용합니다. 

2) 스토리 포인트의 최대치를 정합니다. (13 포인트) 

3) 스토리 포인트 기준 값을 정합니다. (중간값 3포인트나 기본 포인트인 1포인트 등) 4) 스토리 포인트 추정 작업 진행 (다양한 방법으로 진행 가능) 

4-1) 플래닝 포커(planning poker) 기법 : 사용자 스토리를 설명한 후에 각자 해당 사용자 스토리에 어느 정도 포인트를 할당할지 생각한 뒤 해당 값이 적혀 있는 카드를 선택하거나 종이에 그 값을 적은 후 동시에 공개해 가장 많이 나온 값으로 스토리 포인트를 할당하는 기법입니다. 의견이 갈릴 경우 각자의 의견을 듣고 다시 포커를 진행해 스토리 포인트를 추정하는 작업을 반복합니다. 

5) 팀원들이 해당 스프린트에 사용할 스토리 포인트를 정합니다. (15 포인트) 6) 스프린트 종료 후 획득한 스토리 포인트(10 포인트)를 확인하고 계획한 스토리 포인트 보다 적은 스토리 포인트를 얻은 경우 둘의 중간값(12~13 포인트)을 다음 스프린트의 스토리 포인트로 정합니다.
